Before You Start

Prerequisites

Make sure you have completed the Pre-Workshop Setup — Xcode installed, the Rails app running at localhost:3000, and the iOS repo cloned.

Branch structure

The iOS repo has 4 branches that build on each other:

  • main — Steps 1–6: Project setup, Hotwire config, path configuration, environments, HTTP access
  • button-component — Steps 7–8: Switch to local, ButtonComponent bridge
  • new-board-modal — Steps 9–10: Path configuration for modals and replace presentation
  • form-component — Steps 11–12: FormComponent bridge with submit enable/disable

You can check out any branch to see the state of the app at that point: git checkout button-component

Step 2 · Branch: main

Add Hotwire Native Package

Commit: Base iOS app

1 Add the HotwireNative Swift Package

In Xcode:

  1. Go to File → Add Package Dependencies...
  2. Enter the URL: https://github.com/hotwired/hotwire-native-ios
  3. Set version rule to Up to Next Major Version from 1.2.2
  4. Click Add Package
  5. Select HotwireNative library and click Add Package
2 Update SceneDelegate with Navigator

Replace the contents of SceneDelegate.swift with the Hotwire Navigator setup:

HotwireKanban/SceneDelegate.swift 
import HotwireNative
import UIKit

let rootURL = URL(string: "https://hotwire-native-demo.dev")!

class SceneDelegate: UIResponder, UIWindowSceneDelegate {
    var window: UIWindow?

    private let navigator = Navigator(configuration: .init(
        name: "main",
        startLocation: rootURL
    ))

    func scene(_ scene: UIScene, willConnectTo session: UISceneSession, options connectionOptions: UIScene.ConnectionOptions) {
        window?.rootViewController = navigator.rootViewController
        navigator.start()
    }
}
Outcome Your app now loads the Hotwire Native demo site. Run the app to see it in action!

2 Load Path Configuration in AppDelegate

Update AppDelegate.swift to load the path configuration:

HotwireKanban/AppDelegate.swift 
import HotwireNative
import UIKit

@main
class AppDelegate: UIResponder, UIApplicationDelegate {

    func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {

        Hotwire.loadPathConfiguration(from: [
            .file(Bundle.main.url(forResource: "path-configuration", withExtension: "json")!)
        ])

        return true
    }

    // MARK: UISceneSession Lifecycle

    func application(_ application: UIApplication, configurationForConnecting connectingSceneSession: UISceneSession, options: UIScene.ConnectionOptions) -> UISceneConfiguration {
        return UISceneConfiguration(name: "Default Configuration", sessionRole: connectingSceneSession.role)
    }

    func application(_ application: UIApplication, didDiscardSceneSessions sceneSessions: Set<UISceneSession>) {
    }
}
Outcome Path rules now control navigation behavior — URLs ending in /new open as modals.

Step 5 · Branch: main

Environment Support

Commit: introduce Remote and Local url support

1 Create KanbanBoard.swift

Create a new file KanbanBoard.swift to manage environment switching:

HotwireKanban/KanbanBoard.swift 
import Foundation

enum KanbanBoard {
    // Update this to choose which demo is run
    static let current: Environment = .remote

    enum Environment {
        case remote
        case local

        var url: URL {
            switch self {
            case .remote:
                return URL(string: "https://hotwire-native-demo.dev")!
            case .local:
                return URL(string: "http://localhost:3000")!
            }
        }
    }
}
2 Update SceneDelegate to use KanbanBoard

Update the navigator to use the environment URL:

HotwireKanban/SceneDelegate.swift 
import HotwireNative
import UIKit

class SceneDelegate: UIResponder, UIWindowSceneDelegate {
    var window: UIWindow?

    private let navigator = Navigator(configuration: .init(
        name: "main",
        startLocation: KanbanBoard.current.url
    ))

    func scene(_ scene: UIScene, willConnectTo session: UISceneSession, options connectionOptions: UIScene.ConnectionOptions) {
        window?.rootViewController = navigator.rootViewController
        navigator.start()
    }
}
3 Add Server-Side Path Configuration

Update AppDelegate to also load path configuration from the server:

HotwireKanban/AppDelegate.swift (update loadPathConfiguration) 
        Hotwire.loadPathConfiguration(from: [
            .file(Bundle.main.url(forResource: "path-configuration", withExtension: "json")!),

Step 6 · Branch: main

Allow HTTP Access

Commit: Allow http access on localhost

1 Update Info.plist for Local Networking

iOS blocks HTTP connections by default (App Transport Security). To connect to your local Rails server, update Info.plist:

HotwireKanban/Info.plist (add to dict) 
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsLocalNetworking</key>
    <true/>
</dict>

Or in Xcode: select Info.plist, add a new row App Transport Security Settings, and set Allow Local Networking to YES.

Outcome Your app can now connect to http://localhost:3000 for local development.

Step 8 · Branch: button-component

ButtonComponent Bridge

Commit: ButtonComponent Swift component added

1 Create Bridge Group

In Xcode, create a new group called Bridge inside your HotwireKanban folder to organize bridge components.

2 Create ButtonComponent.swift

Create ButtonComponent.swift in the Bridge group:

HotwireKanban/Bridge/ButtonComponent.swift 
import HotwireNative
import UIKit

final class ButtonComponent: BridgeComponent {
    nonisolated override static var name: String { "button" }

    private var barButtonItem: UIBarButtonItem?

    private var viewController: UIViewController? {
        delegate?.destination as? UIViewController
    }

    override func onReceive(message: Message) {
        let event = message.event

        switch event {
        case "connect":
            handleConnectEvent(message: message)
        default:
            print("ButtonComponent: Unknown event \(event)")
        }
    }

    private func handleConnectEvent(message: Message) {
        guard let data: MessageData = message.data() else { return }
        showToolbarButton(title: data.title)
    }

    private func showToolbarButton(title: String) {
        guard let viewController = viewController else { return }

        let action = UIAction { [weak self] _ in
            self?.performButtonClick()
        }

        barButtonItem = UIBarButtonItem(title: title, primaryAction: action)
        barButtonItem?.tintColor = UIColor(red: 1.0, green: 0.4, blue: 0.0, alpha: 1.0) // #FF6600

        viewController.navigationItem.rightBarButtonItem = barButtonItem
    }

    private func performButtonClick() {
        reply(to: "connect")
    }
}

private extension ButtonComponent {
    struct MessageData: Decodable {
        let title: String
    }
}
3 Register ButtonComponent

Update AppDelegate.swift to register the bridge component:

HotwireKanban/AppDelegate.swift (add before return true) 
        // Register bridge components
        Hotwire.registerBridgeComponents([
            ButtonComponent.self
        ])
Outcome The ButtonComponent receives messages from the web and displays a native toolbar button. When tapped, it sends a reply back to the Stimulus controller.

Step 12 · Branch: form-component

FormComponent Implementation

Commit: FormComponent implemented

1 Complete FormComponent Implementation

Replace the scaffold with the full implementation:

HotwireKanban/Bridge/FormComponent.swift 
import HotwireNative
import UIKit

final class FormComponent: BridgeComponent {
    nonisolated override static var name: String { "form" }

    private var submitBarButtonItem: UIBarButtonItem?

    private var viewController: UIViewController? {
        delegate?.destination as? UIViewController
    }

    override func onReceive(message: Message) {
        let event = message.event

        switch event {
        case "connect":
            handleConnectEvent(message: message)
        case "submitEnabled":
            handleSubmitEnabled()
        case "submitDisabled":
            handleSubmitDisabled()
        default:
            print("FormComponent: Unknown event \(event)")
        }
    }

    private func handleConnectEvent(message: Message) {
        guard let data: MessageData = message.data() else { return }
        showToolbarButton(title: data.submitTitle)
    }

    private func showToolbarButton(title: String) {
        guard let viewController = viewController else { return }

        let action = UIAction { [weak self] _ in
            self?.performSubmit()
        }

        submitBarButtonItem = UIBarButtonItem(title: title, primaryAction: action)
        submitBarButtonItem?.tintColor = UIColor(red: 1.0, green: 0.4, blue: 0.0, alpha: 1.0) // #FF6600

        viewController.navigationItem.rightBarButtonItem = submitBarButtonItem
    }

    private func handleSubmitEnabled() {
        submitBarButtonItem?.isEnabled = true
    }

    private func handleSubmitDisabled() {
        submitBarButtonItem?.isEnabled = false
    }

    private func performSubmit() {
        reply(to: "connect")
    }
}

private extension FormComponent {
    struct MessageData: Decodable {
        let submitTitle: String
    }
}
2 Register FormComponent

Update AppDelegate.swift to register both components:

HotwireKanban/AppDelegate.swift 
        // Register bridge components
        Hotwire.registerBridgeComponents([
            ButtonComponent.self,
            FormComponent.self
        ])
Outcome The FormComponent handles form submissions with a native toolbar button. It enables/disables based on form validation and submits the form when tapped.
